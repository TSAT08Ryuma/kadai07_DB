# ①課題名
楽しかったことを記録するツール

## ②課題内容（どんな作品か）
- 登録画面→登録処理画面→登録内容確認画面に遷移。
- ボタンをクリックすると楽しかったことを記録できるようにしました。
- ★を選んでおすすめ度も入れられます

- ID: NA
- PW: NA

## ⑤工夫した点・こだわった点
- データベース名・ユーザー名・パスワードなどの接続情報を 公開ディレクトリの外（dbフォルダ）に分離し、直接URLから参照できない構成にしました。
- ローカル環境（XAMPP）と本番環境（さくらサーバ）で PHPコードを変えずに動かせるよう、DB接続設定を1か所にまとめました。
- ID/PASSなどが入ったコードはアップロードしていないです。

## ⑥難しかった点・次回トライしたいこと（又は機能）
- 本番環境での ディレクトリ構成とURLの対応関係を理解するのが難しかったです（www がURLに含まれない点など）。
- require_once の相対パスがローカルと本番でずれて 500エラーが発生した原因特定に時間がかかりました。

## ⑦フリー項目（感想、シェアしたいこと等なんでも）
- エラーにかなり詰まりましたが、1つずつ原因を切り分けて直していくことで、PHPとサーバの仕組みへの理解が深まったと感じています。
